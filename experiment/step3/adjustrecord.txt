只加了一行，位置在路径常量之后、所有 import 之前：

python

os.environ["PROJECT_ROOT"] = PROJECT_ROOT

为什么必须在 import 之前：diffcsp/common/utils.py 的第 92 行是模块级代码 PROJECT_ROOT: Path = Path(get_env("PROJECT_ROOT"))，Python 第一次 import 任何 diffcsp.* 子模块时就会执行它。如果环境变量还没设，立刻抛 KeyError。把 os.environ 赋值放在所有 import 之前，就能保证 utils.py 被加载时已经能读到这个值。还没有，这是一个真实的错误。Windows 上用 num_workers > 0 启动多进程时，必须有 if __name__ == '__main__': 保护，否则子进程会尝试重新导入主脚本，触发这个 RuntimeError。
