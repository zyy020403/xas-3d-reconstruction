## §1 错误声明

EXPERIMENT4_FINAL_REPORT.md §0.3 + §4.1 写:

> "Phase 6 五子全 PASS"
> "重跑 forward_test.py: **5/5 PASS, 18.1 s wall**"
> "6.5 Full forward+backward | **GPU** | **fp32** | **PASS**"

**这条声明源自 SA4-续 2 的最终报告,MA5 接手时未独立验证**。

## §2 实际情况

Exp5 SA1 在 2026-04-28 复现 Phase 6.5 时发现:

- **真实路径**: PyTorch 2.4.1+cu124 下,forward_test.py Phase 6.5 (GPU bf16) **应该 fail**,因为存在 3 处 hardcoded fp32 与 model bf16 weights 产生 mat1/mat2 dtype mismatch:
  1. `SinusoidalTimeEmbeddings` 内部 `torch.arange(half_dim)` 默认 fp32 → `time_emb` fp32 不 follow model bf16
  2. `t_per_atom` 沿用 `time_emb` 的 fp32
  3. `cspnet` 无 dtype-aware,`atom_latent_emb` (weight bf16) cat (node_features bf16, t_per_atom fp32) 后第二处 mismatch

- **SA4-续 2 报告的 PASS 可能性**(MA5 排序,从最可能到最不可能):
  1. forward_test.py 6.5 测试逻辑漏洞,异常被吞,统计错报 PASS
  2. 跑的是 .bak/.bak2/.bak3 中的别的版本
  3. PT 2.4.1 在某些 hardware 状态下宽容(低概率)
  4. SA1 现在诊断错(低概率,SA1 路径分析具体到行号 + dtype 推导链条)

## §3 影响评估

### 3.1 不影响项

- **Exp4 训练结果完全不受影响**: SA2 训练全程使用 fp32(MA4 决策 D1),Phase 6.5 测的是 bf16 path,**不在 Exp4 生产路径上**。
- **Exp4 best ckpt (val_loss=0.7300) 完全可信**: 训练路径 = fp32,与 Phase 6.4 (CPU fp32 PASS) 路径同 dtype。
- **Step 5 评估结果完全可信**: sample 也是 fp32 推理。
- **Exp5 fine-tune 起点完全可信**: 继承 fp32 即可。

### 3.2 影响项

- **Exp4 final report 5/5 PASS 表述需修正为**: "5/5 PASS in fp32 production path; Phase 6.5 (GPU bf16) skipped — see Errata"
- **未来若想试 bf16 加速训练**: 必须先修 3 处 hardcoded fp32(forward path 内 + cspnet 内),不能直接套用 Exp4/Exp5 训练脚本

## §4 已采取的修正措施

### 4.1 Exp5 SA1 决策(MA5 拍板 B,2026-04-28)

forward_test.py Phase 6.5 改造为 **skip-with-note**:
```python
def phase_65_gpu_bf16_or_skip():
    print("[phase 6.5] SKIPPED (by design): bf16 GPU path not in current "
          "production. See OUTPUT.md §5.7 for rationale.")
    return "SKIPPED"
```

phase counter 改 **5/5 PASS + 1 skipped-by-design**(不是 6/6,skipped ≠ PASS)。

### 4.2 不动 Exp4 服务器代码

Exp4 项目已 closed,不回头改 Exp4 forward_test.py。Exp5 用自己的 forward_test 复制 + skip 即可。

### 4.3 Patch 1 保留

SA1 已加的 `.to(c0.dtype)` 保留——fp32 下 bit-exact 等价(c0 fp32 → 等价 `.float()`),零风险,为未来 bf16 path 留起点。

## §5 Lessons Learned

1. **MA 接手交接时必须独立验证关键 PASS 声明**: SA4-续 2 报告的 PASS,MA4→MA5 transition 时应该 require 实际 console log 原文,而不是接受汇总表。
2. **接力链文档需要"PASS 类型"区分**: 简单 5/5 PASS 不够,需细化为"全 PASS / N PASS + M skipped-by-design / N PASS + M failed-non-blocking"。
3. **测试要反映真实生产路径**: Phase 6.5 测的是不会被走的 path,即使真 PASS 也低价值;skip-with-note + 行号引用比静默 PASS 更诚实。
4. **下游 final report 引用历史声明时**: 应至少标注"基于 SA-X 报告,未独立复核"——MA5 写 final report §0.3 应加这个 caveat。
