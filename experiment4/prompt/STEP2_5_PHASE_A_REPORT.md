> **⚠⚠ FLAG**：`n_center_sites >= 2` 占比 = **89.86%**，是 handoff §5.2 的 20% 触发阈值的 **4.5 倍**。

### 这个 flag 的含义
Handoff §5.2 原文："若 `n_center_sites >= 2` 的样本比例 > 20%，汇报里标注——这提示'第一位点'可能对大量样本不是真正的 FEFF 计算位点，Step 3 的 Dataset 可能需要更精细策略。"

实际占比 89.86% 意味着：**几乎每个样本在原胞里都有多个该元素的位点，选第一个的确定性策略在 Step 3 Dataset 阶段大概率会引入"位点不匹配"噪声**。例如 4-site O 样本，FEFF 实际可能对位点 2 或 3 算谱，但我们用位点 0 的邻居环境去标注 target——label 与谱脱钩。

### 本步仍然选"第一位点"（LOCKED 未变）
对 Phase A / Phase B 输出的壳层边界，"第一位点"策略足够用——因为壳层边界统计是对样本级结构的聚合特征，不强依赖"正确位点"。

### Step 3 需要 MA 裁决的方向（此处只列选项，不做决定）

| 方案 | 代价 | 效果 |
|---|---|---|
| **A**. 维持"第一位点" | 零成本 | 90% 样本 label 可能与谱微偏 |
| **B**. 对每个样本遍历所有等价位点，取"shell 结构最代表"的一个 | O(n_sites) × 邻居计算 | 需要定义"代表性" |
| **C**. 对每个样本产生 k 个（位点, shell_boundary）子样本，训练时随机选 | 数据量 × k | 扩充数据 + 隐式平均 |
| **D**. 用 `SpacegroupAnalyzer.get_symmetrized_structure()` 找 Wyckoff 唯一位点，每个 mp_id 对每个中心元素只保留一份独立位点 | 中等 | 对称性合法的"平均"处理 |

**建议 MA 在 Step 3 启动前决策**。Phase A / Phase B 不做此处理。
