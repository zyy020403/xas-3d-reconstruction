## 第一部分：指标怎么算的，数字什么意思

在看结果之前，先弄清楚我们在量什么。

### 1.1 RMSD 是什么

RMSD（均方根偏差，Root Mean Square Deviation）是衡量"预测的原子位置离真实位置有多远"的指标，单位是埃（Å，1Å = 0.1 纳米）。

**计算过程**是这样的：

我们有 20 个真实原子位置，和 20 个模型预测的原子位置。问题是：预测的哪个对应真实的哪个？我们用一种叫做"匈牙利匹配"的算法，找到让总距离最小的一一对应关系，然后计算每对之间距离的平均值。

举个具体例子：  
假设有 5 个原子（简化），真实位置在 [A, B, C, D, E]，预测位置在 [A', B', C', D', E']。匈牙利匹配找到最优配对，比如 A→A'（差 0.5Å）、B→C'（差 1.2Å）……把所有差距平方后平均再开根，就是 RMSD。

**参考标准：**
- 随机基线：如果把 20 个原子随机撒在 L=6Å 的盒子里，RMSD 大约是 **2.32 Å**
- 我们的结果：**1.47 Å**，比随机基线好 37%
- 物理含义：预测的原子位置平均离真实位置差 1.47Å，大约是一个 Fe-O 键长（~2.0Å）的 73%

### 1.2 Type Accuracy 是什么

Type Accuracy（原子类型准确率）是衡量"预测的原子类型（哪种元素）对不对"的指标。

**计算过程**：在匈牙利匹配配好对之后，检查每一对"真实原子 vs 预测原子"是不是同一种元素，正确的比例就是 Type Accuracy。

**关键背景**：Fe 氧化物体系里，邻居原子可能是 O、Fe、Ca、Mn、Ti、Si 等几十种元素。完全随机猜的准确率约 **1%**（1/100）。

**我们的结果 25% 意味着**：在 20 个邻居原子里，平均有 5 个元素类型被猜对了。比随机猜好 **25 倍**。

**为什么不高**：XANES 谱对元素类型的区分能力本来就有物理限制。谱图能告诉你"附近大概是氧化物环境"，但很难区分"第三壳层是 Ca 还是 Na"。这是物理信息量的天花板，不是模型能力不够。

### 1.3 pred_in_cutoff / true_in_cutoff 是什么

这两个指标回答一个问题：**预测出来的原子有没有出现在 Fe 附近，而不是散布在整个盒子里？**

- `true_in_cutoff`：真实结构中，距 Fe 原点 ≤ eval_cutoff 范围内有多少个原子。我们的结果约 17-19/20，说明真实原子确实聚集在 Fe 附近（符合物理常识）。
- `pred_in_cutoff`：预测的原子中，落在 Fe 附近的有多少个。

**问题所在**：在早期版本（Step4/4b/4c）中，pred_in_cutoff 只有约 2.8/20，而 true 是 17/20。这说明模型预测的原子散布在整个盒子里，根本没有聚集在 Fe 附近，等同于随机撒点。修复 L=6+min-image 之后，pred_in_cutoff 提升到 17.5/20，与真实值接近，说明模型终于学会了"原子应该在 Fe 附近"这件事。

## 第二部分：实验完整历史

### 2.1 任务定义

输入：一条 Fe K-edge XAS 谱（XANES 区间 150 点 + EXAFS chi1 200 点 + 73 维物理先验特征）  
输出：以 Fe 为中心的最近 20 个邻居原子的类型（原子序数）和位置（坐标）  
模型：改造后的 DiffCSP 扩散模型

**扩散模型是什么**：扩散模型的思路是"先把数据加噪到随机分布，再训练模型学会去噪"。类比：给一张照片加雪花噪声，直到完全看不出原图；然后训练模型"给定谱图作为条件，把雪花噪声一步步去掉，还原出原子结构"。

### 2.3 各版本训练记录

#### Step4（原始版）— 2026-04-09，废弃

**配置**：L=12Å，坐标系 [-0.5, 0.5]，但 forward() 有 `% 1.`  
**训练**：val_loss=0.6178，epoch 489，收敛正常  
**评估**：RMSD=4.51Å（随机基线 4.65Å），pred_in_cutoff=2.75/20  
**结论**：**坐标系不一致 bug**。Dataset 输出 [-0.5, 0.5]，但 forward() 用 `% 1.` 强制折叠到 [0,1]，导致原子坐标人为撕裂成双峰分布，模型学到的是错误的分布。废弃。

#### Step4b — 2026-04-09，废弃

**改动**：Dataset 加 `% 1.0`，使两端统一为 [0,1]  
**训练**：val_loss=0.6504，epoch 494，收敛正常  
**评估**：RMSD=4.17Å，pred_in_cutoff=2.89/20  
**结论**：坐标系不一致修复了，但 [0,1] 表示对本任务天然不利。原子真实分布集中在原点附近（Fe 中心），而 DiffCSP 的扩散先验是 [0,1] 均匀分布。用 `% 1.0` 折叠后，"聚集在原点的原子"变成了"一部分在 0 端，一部分在 1 端"的双峰，模型依然无法学习。

**什么叫双峰**：想象把一根橡皮筋绕在圆柱上，原本集中在一处的点，被"折叠"后出现在圆柱两端，看起来像两个独立的堆。这就是双峰。

#### Step4c — 2026-04-09，废弃

**改动**：去掉 forward() 中的 `% 1.`，Dataset 回退到 v5（[-0.5, 0.5]）  
**训练**：val_loss=0.65，epoch ~250  
**评估**：RMSD=4.20Å，pred_in_cutoff=2.77/20  
**结论**：两端坐标系统一了，但 L=12Å 导致原子占 box 体积仅 3%，先验空间太大。诊断实验（reverse diffusion std 曲线）显示：模型中间阶段（t=640→420）确实捕捉到信号（std 从 0.30 降到 0.25），但信号出现时原子已经散布在离真实位置 4-5Å 之外，剩余低噪声步骤步长不足以收敛。

**什么叫先验不匹配**：扩散模型需要从"均匀撒满整个盒子"的初始状态，经过很多步去噪，收敛到"原子聚集在中心 3% 空间"。这就像从一锅均匀的浓汤里，只靠微弱的化学信号，把所有盐粒引导到锅底中央——信号存在，但距离太远，步长太小，完不成。

#### Step4d — 2026-04-09，**最终有效版本**

**改动**：L=12Å → L=6Å；Dataset 加 `frac -= np.round(frac)`（min-image 折叠）

**什么是 min-image 折叠**：坐标除以 L 后，加上"减去最近整数"这一步，把所有原子折叠回 [-0.5, 0.5]。这是晶体学的标准做法，确保每个原子都取"到原点最近的镜像"。举例：4Å 处的原子在 L=6 时，frac=4/6=0.67，减去 round(0.67)=1，得到 -0.33，在 [-0.5, 0.5] 内。

**为什么 L=6 有效**：L=6 后，原子占 box 体积从 3% 提升到约 25%。中间扩散阶段信号出现时，原子离真实位置的平均距离减半，剩余步骤的步长完全足够收敛。

**训练**：val_loss=0.8554，epoch 249，收敛  
**评估（val/test）**：RMSD=1.47Å，Type Accuracy=0.249，pred_in_cutoff=17.47/20 ✅

## 第三部分：Experiment 3 改进 Proposal

### 3.1 当前版本的上限在哪里

Experiment 2 的 Type Accuracy 在 epoch 100 左右就停止下降了。继续训练对坐标有帮助，但对原子类型预测没有帮助。这说明：**当前的 SpectrumEncoder + 扩散 decoder 的架构，已经把谱图中关于原子类型的信息榨干了，剩下的误差是物理信息量的天花板，不是训练不足的问题**。

具体原因：原子类型预测是扩散模型的"附带任务"，和坐标预测共享同一个 decoder，优化目标存在竞争。谱图对类型的约束本来就比对距离的约束弱（EXAFS 的振荡频率对键长敏感，但对"到底是 Ca 还是 Na"不太敏感）。

### 3.2 两个改进方向

---

#### 方向 A（推荐，改动小，预期效果显著）：独立的原子类型分类 Head

**思路**：把"预测原子类型"从扩散任务里解耦出来，给它单独的一条路径。

**具体做法**：
1. SpectrumEncoder 输出的 latent 向量（256维）之后，额外加一个 MLP 分类器
2. 这个分类器直接预测"第 k 近的邻居是什么元素"（多分类问题）
3. 用交叉熵损失单独优化，与扩散 loss 加权合并

**预期提升**：文献中有专门做这件事的工作（Zheng et al., 2020，npj Computational Materials），用随机森林从 XANES 预测配位环境，准确率 60-80%。我们有更好的模型和更大的数据集，Type Accuracy 应该可以从 25% 提升到 40-60%。

**改动范围**：只需修改 `diffusion_w_type_xas.py`，加一个 MLP head 和对应的 loss 项，其余不变。

---

#### 方向 B（进阶，改动大，潜力更高）：引入外部预训练 XAS 模型

**关键发现**：有一个来自 Materials Project 官方团队的开源库，专门做"从 XAS 谱预测配位环境"，且与我们的数据完全兼容：

**`maml`（MAterials Machine Learning）**
- GitHub：`https://github.com/materialsvirtuallab/maml`
- PyPI：`pip install maml`
- 核心模块：`maml.apps.rfxas`（Random Forest XAS）

这个库里的 `rfxas` 模块专门用随机森林从 XANES 谱预测原子局部环境（配位数、键长、Bader 电荷），与我们的任务高度相关。它由 UC San Diego 的 Ong 课题组开发，训练数据来自 Materials Project 的 80 万条 FEFF 计算的 XANES，包含 Fe K-edge，与我们的数据源相同。

**具体使用方式**：

Step 1：用 `maml.apps.rfxas` 在我们的训练集上训练一个 XAS→元素类型分类器，输入是 xmu_xanes（150点），输出是配位元素种类的概率分布。

Step 2：把这个分类器的输出作为额外条件，拼接进 SpectrumEncoder 的 latent 向量里（condition 维度从 256 → 256+N_classes）。

Step 3：这样扩散模型收到的条件信号里，已经包含了"这个位置大概是什么元素"的先验猜测，去噪时可以直接利用这个信息。

**预期效果**：Type Accuracy 可能从 25% 提升到 50-70%。

## 附录：关键概念解释

**笛卡尔坐标** vs **分数坐标**：笛卡尔坐标是普通的 XYZ 坐标（单位 Å）。分数坐标是"相对于盒子大小的比例"——如果盒子边长 L=6Å，一个原子在笛卡尔坐标 3Å 处，分数坐标就是 3/6=0.5。我们训练时用分数坐标，因为扩散模型对 [0,1] 或 [-0.5, 0.5] 范围的数处理更稳定。

**双峰分布**：原子聚集在 Fe 原点附近，笛卡尔坐标范围约 ±3Å，分数坐标范围约 ±0.5（L=6 时）。如果对这些分数坐标做 `% 1`，-0.3 会变成 0.7，0.3 还是 0.3——原本连续的分布被撕成两段，一段在 [0, 0.3]，一段在 [0.7, 1.0]，中间 [0.3, 0.7] 空着，就是双峰。

**最小镜像（min-image）**：晶体结构是周期性的，每个原子有无数个"镜像"（每隔一个盒子重复一次）。最小镜像就是"取距离最近的那个镜像"。技术上就是 `frac -= round(frac)`，把坐标从任意范围折叠回 [-0.5, 0.5]。在匈牙利匹配里也要用最小镜像，因为越过盒子边界的原子对，直接算距离会偏大。

**先验不匹配**：扩散模型从"均匀分布"开始去噪。如果真实数据集中在很小的空间（如中心 3%），模型需要从 97% 的"空旷区域"去噪到 3% 的"真实区域"，信噪比极低。缩小盒子（L=12→L=6）是解决这个问题最直接的方法。
